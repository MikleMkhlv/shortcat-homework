# Ответы на вопросы

1. В первом примере, где не используются примипивы синхронизауии происзодит data race т.к. каждая созданная гоурутина (1000 шт) обращается к оному участку памяти (переменная counter). Из-за этого при каждом инкременте переменной, горутина в этот момент знает одно значение и инкрементирует его, а параллельно другая горутина пытается одновить счетчик и у него другое значене. Сооотв. из-за этого мы не можем получить 1000 записей. В отсальных случаеях data race исправляется за счет мьбтекса или атомарной операции.
2. По поводу сравнения скорости. в каждую функцию добавил точки начала и конца работы. По ним видно, что время выполнения фунуции с атомиком на 2мс быстрее, чем с мьютессом т.к. не выполняется стандартный инкремент (чтение -> изменение -> запись), а делается запись в один шаг на уровне процессора.
